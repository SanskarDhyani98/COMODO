The problem

Egocentric Human Activity Recognition (HAR) wants to know what a person wearing a body-mounted device is doing — walking, cooking, climbing stairs.

Cameras (video) are accurate but power-hungry, privacy-invasive, and impractical for always-on wearables. IMU sensors (accelerometer + gyroscope) are cheap, private, and battery-friendly — but their data is hard to label and small in quantity.

The COMODO idea

Use a large pretrained video model as a silent teacher and distil its knowledge into a small IMU student — without any labels.

  • 📹 Teacher: VideoMAE / TimeSformer (frozen).
  • 📈 Student: Mantis-8M / MOMENT time-series model.
  • 🔁 Trick: a FIFO queue of video embeddings keeps the reference distribution stable while we distil.
  • 🏷️ No labels needed — pure self-supervised cross-modal transfer.

What the COMODO loss does

For each (video, IMU) pair we compute embeddings zv and zx. The student is trained so that its similarity distribution over a queue of past video embeddings matches the teacher's similarity distribution over the same queue. 

ℒ = − Σᵢ pTᵢ · log pSᵢ
where  pT = softmax( zv·Qᵀ / τt )
       pS = softmax( zx·Qᵀ / τs )

Our six inference-time improvements

The COMODO paper trains a strong IMU student, but the way you use it at inference matters too. None of these need retraining — they all run on top of the same frozen Mantis-8M student we loaded.

1

Multi-prototype matching

Compare to K=5 reference embeddings per class instead of one. Reduces variance — one unlucky reference no longer poisons the class.

2

Test-Time Augmentation

Run the model on N=8 augmented copies of the input (noise + time-shift + amplitude jitter) and average the embeddings. Standard trick from image classification.

3

Noise-robustness sweep

Inject Gaussian noise (loose strap, vibration) into the IMU and measure accuracy. Proves the improvements are real, not cosmetic.

4

Channel saliency

Gradient × input tells us which IMU channel mattered most. Turns the black box into something a wearable engineer can debug.

5

Batched inference

Building reference embeddings: 30 sequential forward passes vs 1 batched pass. Same outputs, much lower latency.

6

Confidence calibration

Convert raw cosine similarities to a softmax probability ("87% running") via a temperature-scaled softmax. Non-ML viewers can finally read the output.

③ — Robustness under sensor noise

We sweep added Gaussian noise σ from 0 → 1.2 sensor-units and measure classification accuracy across all 6 activities. Query signals also use ±15% parameter perturbation to simulate person-to-person variation. Each point averages — queries.

idle

The story is the shape of the curve: the system stays at ceiling until σ≈0.2, degrades gracefully through σ=0.5, then collapses toward chance (≈16% for 6 classes) at σ=1.2 — confirming the Mantis-8M features carry real activity information rather than just memorising the prototypes. On this synthetic data the four inference-time variants land within noise of one another (improvements show up most clearly on real Ego4D / EgoExo4D — see Benchmark results tab).

① + ② — Before vs After: single noisy query

Run the same noisy IMU window through all four configurations and see whether the prediction is correct. The killer demo case: cycling, seed 22, noise σ=0.7 — baseline confuses it with "shaking", and every improvement flips it to the correct answer.

How the upload flow works (end-to-end, in 6 steps)

  1. Browser uploads the CSV as multipart/form-data to POST /api/infer-upload.
  2. Server parses the text — header lines are auto-detected and skipped. Columns / rows can be either orientation (we autodetect which dim is 6).
  3. The (6, M) array is linearly interpolated to (6, 1000) so it matches the Mantis-8M-compatible window length.
  4. The student model (already loaded into memory at warmup) does one forward pass on CPU, ~20 ms.
  5. The output 128-d embedding is L2-normalised and compared against either 1 (baseline) or K=5 (improved) reference embeddings per activity via cosine similarity.
  6. Cosines are passed through a temperature-scaled softmax → calibrated probability distribution → top-1 wins.
